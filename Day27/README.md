# Day 27 – Linux Help and Documentation Practice Lab

**Date:** 4th June, 2026

## Summary

Today I practiced using Linux help and documentation commands. I learned how to find information about commands, read manual pages, and understand command options without searching the internet. This is an important skill for Linux administrators because most answers can be found directly from the system documentation.

### Commands Practiced

* man ls
* man pwd
* man mkdir
* man touch
* --help
* whereis
* which
* whatis

### Key Learning

* Learned how to use manual pages (`man`) to understand commands.
* Used `--help` to view command options quickly.
* Practiced locating command binaries with `which` and `whereis`.
* Learned how `whatis` provides a brief description of commands.
* Improved my ability to troubleshoot and learn independently.

### Reflection

This practice lab showed me that Linux provides extensive built-in documentation. Instead of memorizing every command, I can use tools such as `man`, `whatis`, and `--help` to quickly find information when needed. This is a valuable skill for anyone pursuing Linux, System Administration, or DevOps.
